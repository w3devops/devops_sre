üöÄ Hands-On Practice: Velero Backup

üì¶ Step 1: Install Velero

Install Velero for cluster backup and restore:

Install Velero CLI:

wget https://github.com/vmware-tanzu/velero/releases/download/v1.12.0/velero-v1.12.0-linux-amd64.tar.gz
tar -xzf velero-v1.12.0-linux-amd64.tar.gz
sudo mv velero-v1.12.0-linux-amd64/velero /usr/local/bin/

Install Velero server:

velero install \
  --provider aws \
  --plugins velero/velero-plugin-for-aws:v1.8.0 \
  --bucket my-bucket \
  --backup-location-config region=us-east-1 \
  --snapshot-location-config region=us-east-1

kubectl apply -f practice/backup.yaml

üìä Step 3: Resource Optimization

Optimize resource usage and cluster performance:

Check resource utilization:

# Install metrics server
kubectl apply -f https://github.com/kubernetes-sigs/metrics-server/releases/latest/download/components.yaml

# Check node resources
kubectl top nodes

# Check pod resources
kubectl top pods --all-namespaces

# Identify resource requests/limits
kubectl describe nodes | grep -A 10 "Allocated resources"

Resource optimization strategies:

# Right-size deployments kubectl patch deployment myapp -p '{"spec":{"template":{"spec":{"containers":[{"name":"myapp","resources":{"requests":{"cpu":"100m","memory":"128Mi"},"limits":{"cpu":"500m","memory":"512Mi"}}}]}}}' # Use LimitRanges for namespaces apiVersion: v1 kind: LimitRange metadata: name: resource-limits spec: limits: - type: Container default: cpu: 100m memory: 128Mi defaultRequest: cpu: 50m memory: 64Mi

üîç Step 4: Cluster Troubleshooting

Master essential troubleshooting techniques:

Common issues and solutions:

# Check cluster health
kubectl get cs
kubectl get nodes
kubectl get pods --all-namespaces -o wide

# Check resource constraints
kubectl describe node  | grep -A 5 "Conditions"

# View pod events
kubectl get events --sort-by=.metadata.creationTimestamp

# Check logs across multiple pods
kubectl logs -l app=myapp --tail=100

# Debug network issues
kubectl exec -it  -- nslookup kubernetes.default
kubectl exec -it  -- curl -k https://kubernetes.default/api

üîÑ Step 5: Cluster Maintenance

Perform routine cluster maintenance tasks:

Node maintenance:

# Drain node for maintenance
kubectl drain  --ignore-daemonsets --delete-emptydir-data

# Mark node as unschedulable
kubectl cordon 

# Make node schedulable again
kubectl uncordon 

# Delete node from cluster
kubectl delete node

Certificate rotation:

# Check certificate expiration
kubectl get certificates -A

# Rotate certificates (kubeadm)
sudo kubeadm certs renew all
sudo systemctl restart kubelet

üí∞ Cost Optimization

üí∏ Step 6: Cost Management

Implement cost optimization strategies:

Resource efficiency:

# Find unused resources
kubectl get deployments -A -o json | jq -r '.items[] | select(.spec.replicas == 0) | .metadata.name'

# Identify over-provisioned pods
kubectl top pods --all-namespaces --sort-by=cpu

# Check persistent volume usage
kubectl get pv -o json | jq -r '.items[] | select(.spec.claimRef == null) | .metadata.name'

Cost optimization tools:

# Kubecost for cost monitoring helm repo add kubecost https://kubecost.github.io/cost-analyzer/ helm install kubecost kubecost/cost-analyzer --namespace kubecost --create-namespace # Goldilocks for resource recommendations helm repo add fairwinds-stable https://charts.fairwinds.com/stable helm install goldilocks fairwinds-stable/goldilocks --namespace goldilocks --create-namespace

üèóÔ∏è Multi-Cluster Commands

üíæ Backup & Recovery Commands

# Create backup
velero backup create production-backup --include-namespaces production,monitoring

# List backups
velero backup get

# Restore from backup
velero restore create --from-backup production-backup

# Schedule backups
velero schedule create production-daily --schedule="0 2 * * *"
